Manejo robusto de errores con excepciones

Hacer que tus programas sean robustos significa que pueden manejar situaciones inesperadas sin bloquearse. Python utiliza **excepciones** para señalar errores y el bloque `try/except` para capturarlos y gestionarlos de forma elegante. Esto es crucial cuando se trabaja con operaciones de E/S (entrada/salida) como archivos o redes.

1. Manejo de `FileNotFoundError`

Este error ocurre cuando intentas acceder a un archivo que no existe. Podemos usar `try/except` para informar al usuario en lugar de que el programa falle.
